# Acknowledgments

Cadence Guard builds on the work of many contributors across the Flow ecosystem.

## Flow Engineering Team

The [Cadence Smart Contract Audit Prompt](https://docs.google.com/document/d/1Pk5aaxrpuz1LcXI_GywLpDa2QvX04FkKHdzamOcJ8Yc) developed by the Flow Engineering team informed several checklist items, particularly around code quality, DeFi-specific checks, and Cadence idioms. Thank you for sharing this with the community.

## onflow/cadence-rules

The [cadence-rules](https://github.com/onflow/cadence-rules) repository provides comprehensive AI-friendly development rules for Cadence. Cadence Guard complements that work by focusing on security auditing rather than development guidance. Several patterns and best practices referenced in our checklist align with their rule set.

- [Cadence Security Best Practices](https://cadence-lang.org/docs/security-best-practices)
